# ИС управления заявками для системного администратора (Laravel + MySQL)

Проект реализует серверную часть и web-интерфейс системы управления заявками для ГБПОУ МО СПК на базе Laravel.

## Введение
Цель проекта — централизованная обработка заявок пользователей, безопасное хранение данных и разграничение прав доступа (администратор, менеджер, пользователь).

## Основная часть
Реализация выполнена в структуре Laravel-приложения:
- маршруты: `routes/web.php`;
- контроллеры: `app/Http/Controllers/AuthController.php`, `app/Http/Controllers/TicketController.php`;
- модели: `app/Models/Ticket.php`;
- шаблоны интерфейса: `resources/views/...`;
- точка входа: `public/index.php`.

Текущий функционал:
- вход/выход пользователей;
- просмотр списка заявок;
- создание заявок с приоритетом и статусом;
- базовая валидация входящих данных.

## БД
Используется **MySQL**.

### Подключение к БД
В `.env` должны быть параметры:
- `DB_CONNECTION=mysql`
- `DB_HOST=127.0.0.1`
- `DB_PORT=3306`
- `DB_DATABASE=spk_tickets`
- `DB_USERNAME=spk_user`
- `DB_PASSWORD=secret`

Пример есть в `.env.example`.

### Структура БД (миграции)
- `users` — пользователи системы (`name`, `email`, `password`, `role`, timestamps).
- `tickets` — заявки (`title`, `description`, `priority`, `status`, `created_by`, timestamps).

Связь: `tickets.created_by -> users.id`.

## Сайт
Страницы:
- `/login` — авторизация;
- `/tickets` — список заявок;
- `/tickets/create` — форма создания заявки.

## Заключение
В репозитории подготовлена Laravel-структура сайта и миграции под MySQL. Это база для дальнейшего расширения: комментарии к заявкам, история статусов, API и более детальная ролевая модель.

## Приложение А
Техническая структура репозитория:
- `app/` — бизнес-логика;
- `routes/` — маршрутизация;
- `resources/views/` — Blade-шаблоны;
- `database/migrations/` — структура базы данных;
- `public/` — входная точка web-приложения.

## Приложение Б
Пошаговый запуск:
1. `composer install`
2. `cp .env.example .env`
3. Создать БД MySQL `spk_tickets`
4. `php artisan key:generate`
5. `php artisan migrate`
6. `php artisan serve`
