# Пояснительная записка

**Тема:** Разработка серверной части ИС управления заявками для системного администратора (Laravel + MySQL).

## Введение
Цель проекта — создать серверную часть и сайт для централизованной регистрации, обработки и контроля заявок в ГБПОУ МО СПК.

## Основная часть
Система реализуется на Laravel по MVC-подходу:
- маршруты (`routes/web.php`),
- контроллеры (`app/Http/Controllers/*`),
- модели (`app/Models/*`),
- миграции БД (`database/migrations/*`),
- шаблоны интерфейса (`resources/views/*`).

Реализованные сценарии:
1. Авторизация пользователя.
2. Выход из системы.
3. Просмотр списка заявок.
4. Создание новой заявки с приоритетом.

## БД
Используется MySQL.

Основные сущности:
- `users`: пользователи системы (ФИО/имя, email, пароль, роль);
- `tickets`: заявки (заголовок, описание, приоритет, статус, автор заявки).

Ключевая связь:
- `tickets.created_by -> users.id`.

## Сайт
Страницы web-интерфейса:
- `/login` — вход в систему;
- `/tickets` — список заявок;
- `/tickets/create` — создание заявки.

Валидация данных выполняется средствами Laravel в контроллерах.

## Заключение
Подготовлен базовый рабочий каркас системы, который можно расширять:
- комментарии к заявкам,
- история смены статусов,
- API для клиентской части,
- расширенная ролевая модель и отчеты.

## Приложение А
Структура проекта:
- `app/` — бизнес-логика;
- `routes/` — маршрутизация;
- `resources/views/` — интерфейс (Blade);
- `database/migrations/` — структура БД;
- `public/` — web-точка входа;
- `bootstrap/` — инициализация приложения;
- `artisan` — консольные команды Laravel.

## Приложение Б
Инструкция запуска:
1. `composer install`
2. `cp .env.example .env`
3. Настроить MySQL в `.env`
4. `php artisan key:generate`
5. `php artisan migrate`
6. `php artisan serve`
