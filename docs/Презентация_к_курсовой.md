# Презентация к курсовому проекту

## Слайд 1. Тема проекта
- Разработка серверной части информационной системы управления заявками
- Технологии: Laravel, PHP, MySQL, Blade, Git
- Назначение: централизованный учет и обработка внутренних заявок

## Слайд 2. Актуальность
- Внутренние обращения часто теряются при устной передаче или переписке
- Нет прозрачности статусов и ответственных
- Требуется единая система регистрации и контроля заявок

## Слайд 3. Цель и задачи
- Цель: разработать рабочий сайт для управления заявками
- Задачи:
- спроектировать структуру базы данных
- реализовать аутентификацию и роли
- сделать создание, просмотр и редактирование заявок
- подготовить документацию и демонстрационные материалы

## Слайд 4. Технологический стек
- Backend: PHP 8.2 + Laravel 11
- База данных: MySQL
- Frontend: Blade-шаблоны
- Контроль версий: Git
- React: не использовался, так как проект реализован как server-rendered web-приложение

## Слайд 5. Архитектура проекта
- MVC-подход Laravel
- Маршруты принимают запросы
- Контроллеры обрабатывают логику
- Модели работают с БД
- Представления отображают интерфейс

## Слайд 6. Диаграмма архитектуры
- Пользователь -> Браузер -> Laravel -> MySQL
- Внутри Laravel:
- routes/web.php
- AuthController
- TicketController
- Blade views
- Eloquent models

## Слайд 7. База данных MySQL
- Таблица users
- Таблица tickets
- Таблица sessions
- Связь: tickets.created_by -> users.id
- Реализованы миграции и сидеры

## Слайд 8. Реализованный функционал
- Вход и выход из системы
- Просмотр списка заявок
- Создание заявки только пользователем
- Редактирование существующих заявок администратором и менеджером
- Тестовые данные через DatabaseSeeder

## Слайд 9. Git и история разработки
- Проект подготовлен под размещение в GitHub
- Git нужен для фиксации этапов разработки
- История коммитов подтверждает самостоятельную работу над проектом
- Репозиторий должен содержать код, БД, документацию и презентацию

## Слайд 10. Итоги и развитие
- Создана рабочая основа helpdesk-системы
- Проект пригоден для локального развертывания
- Возможные улучшения:
- комментарии к заявкам
- вложения
- расширенные роли
- API и аналитика

## Слайд 11. Заключение
- Цель проекта достигнута
- Реализован рабочий сайт с БД MySQL
- Подготовлены пояснительная записка и презентация
- Проект можно развивать и использовать как основу для учебной защиты
