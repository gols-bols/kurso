Разработка информационной системы управления заявками

Серверная часть и web-интерфейс для регистрации, отображения и сопровождения внутренних обращений пользователей.

Сергиево-Посадский колледж Слайд 1 / 11
Актуальность

Почему тема важна

  • Устные и разрозненные обращения легко теряются.
  • Пользователь не понимает, на каком этапе находится заявка.
  • Руководитель не видит общую картину нагрузки и исполнения.
  • Нужна единая цифровая точка регистрации и контроля.

Проблема

Без централизованной системы обращения дублируются, теряются и плохо контролируются.

Постановка проблемы Слайд 2 / 11
Цель и задачи

Цель проекта

Разработать рабочий сайт для управления заявками с хранением данных в MySQL и возможностью дальнейшего развития.

  • Спроектировать структуру базы данных.
  • Реализовать авторизацию и роли пользователей.
  • Сделать просмотр, создание и редактирование заявок.
  • Подготовить пояснительную записку и материалы для защиты.

База данных

Основные таблицы

  • users — пользователи и роли
  • tickets — заявки, приоритеты, статусы
  • sessions — пользовательские сессии

Связь

tickets.created_by -> users.id

Структура реализована через миграции Laravel и хранится в MySQL.
